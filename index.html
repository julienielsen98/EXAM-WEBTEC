Recipes API

write "localhost:8080/(an wantent get respons)" in the URL

eksample: localhost:8080/recipes

USAGE USERS:


GET "/recipes"

Returns data stored in table for users


GET "/recipe/:recipe_id"

Returns a chosen recipe by id


GET "/recipe/:recipe_id"

Returns a chosen recipe by id. Details about name, category and ingredients.


GET "/recipe/:recipe_id/all"

Returns a chosen recipe by id, but with all the details.


GET "/recipe/:recipe_id/:step_id_1"

Returns a chosen step by single id. (bug - on step_id_2 and so on...)

USAGE PREMIUM USERS:


GET "/premium/recipes"

Returns data stored in tables for premium user (More recipes options)


GET "/premium/recipe/:recipe_id"

Returns a chosen recipe by id. Details about name, category and ingredients.


GET "/premium/recipe/:recipe_id/all"

Returns a chosen recipe by id, but with all the details.


GET "/premium/recipe/:recipe_id/:step_id_1"

Returns a chosen step by single id. (bug - on get step_id_2 and so on...)


GET "/premium/search/:ingredient1"

Returns all the recipes that contains that ingredient (bug - on get ingredient2 and so on...)


GET "/premium/ingredients"

Returns all the ingredients that exists in the database
